### <a name="working-with-the-orchestration-client"></a><span data-ttu-id="ef413-124">Uso del cliente de orquestación</span><span class="sxs-lookup"><span data-stu-id="ef413-124">Working with the Orchestration client</span></span>

<span data-ttu-id="ef413-125">Las orquestaciones son únicas en comparación con otros estilos de operaciones desencadenadas en Azure Functions.</span><span class="sxs-lookup"><span data-stu-id="ef413-125">Orchestrations are unique when compared to other styles of triggered operations in Azure Functions.</span></span> <span data-ttu-id="ef413-126">Durable Functions permite la ejecución de funciones que pueden tardar horas o incluso días en completarse.</span><span class="sxs-lookup"><span data-stu-id="ef413-126">Durable Functions enables the execution of functions that may take hours or even days to complete.</span></span> <span data-ttu-id="ef413-127">Ese tipo de comportamiento procede de la necesidad de comprobar el estado de una orquestación en ejecución, finalizar de forma preventiva o enviar notificaciones de eventos externos.</span><span class="sxs-lookup"><span data-stu-id="ef413-127">That type of behavior comes with the need to able to check the status of a running orchestration, preemptively terminate, or send notifications of external events.</span></span>

<span data-ttu-id="ef413-128">En estos casos, la extensión Durable Functions proporciona la clase `DurableOrchestrationClient` que permite interactuar con las funciones orquestadas.</span><span class="sxs-lookup"><span data-stu-id="ef413-128">For such cases, the Durable Functions extension provides the `DurableOrchestrationClient` class that allows you to interact with orchestrated functions.</span></span> <span data-ttu-id="ef413-129">Puede acceder al cliente de orquestación mediante el enlace de `OrchestrationClientAttribute`.</span><span class="sxs-lookup"><span data-stu-id="ef413-129">You get access to the orchestration client by using the `OrchestrationClientAttribute` binding.</span></span> <span data-ttu-id="ef413-130">Por lo general, incluiría este atributo con otro tipo de desencadenador, como `HttpTrigger` o `ServiceBusTrigger`.</span><span class="sxs-lookup"><span data-stu-id="ef413-130">Generally, you would include this attribute with another trigger type, such as an `HttpTrigger` or `ServiceBusTrigger`.</span></span> <span data-ttu-id="ef413-131">Una vez que se ha desencadenado la función de origen, se puede usar el cliente de orquestación para iniciar una función de orquestador.</span><span class="sxs-lookup"><span data-stu-id="ef413-131">Once the source function has been triggered, the orchestration client can be used to start an orchestrator function.</span></span>

```csharp
[FunctionName("KickOff")]
public static async Task<HttpResponseMessage> Run(
    [HttpTrigger(AuthorizationLevel.Function, "POST")]HttpRequestMessage req,
    [OrchestrationClient ] DurableOrchestrationClient<orchestrationClient>)
{
    OrderRequestData data = await req.Content.ReadAsAsync<OrderRequestData>();

    string instanceId = await orchestrationClient.StartNewAsync("PlaceOrder", data);

    return orchestrationClient.CreateCheckStatusResponse(req, instanceId);
}
```

### <a name="the-orchestrator-function"></a><span data-ttu-id="ef413-132">La función de orquestador</span><span class="sxs-lookup"><span data-stu-id="ef413-132">The orchestrator function</span></span>

> <span data-ttu-id="ef413-140">*Se ha omitido el código de control de errores por razones de brevedad*</span><span class="sxs-lookup"><span data-stu-id="ef413-140">*Error handling code has been left out for brevity*</span></span>

```csharp
[FunctionName("PlaceOrder")]
public static async Task<string> PlaceOrder([OrchestrationTrigger] DurableOrchestrationContext context)
{
    OrderRequestData orderData = context.GetInput<OrderRequestData>();

    await context.CallActivityAsync<bool>("CheckAndReserveInventory", orderData);
    await context.CallActivityAsync<string>("ProcessPayment", orderData);

    string trackingNumber = await context.CallActivityAsync<string>("ScheduleShipping", orderData);
    await context.CallActivityAsync<string>("EmailCustomer", trackingNumber);

    return trackingNumber;
}
```

<span data-ttu-id="ef413-145">La `Task<string>` completada a partir de `StartNewAsync` debe contener el identificador único de la instancia de orquestación.</span><span class="sxs-lookup"><span data-stu-id="ef413-145">The completed `Task<string>` from `StartNewAsync` should contain the unique ID of the orchestration instance.</span></span> <span data-ttu-id="ef413-146">Este identificador de instancia se puede utilizar para invocar operaciones en esa orquestación específica.</span><span class="sxs-lookup"><span data-stu-id="ef413-146">This instance ID can be used to invoke operations on that specific orchestration.</span></span> <span data-ttu-id="ef413-147">Se puede consultar la orquestación para obtener el estado o enviar notificaciones de eventos.</span><span class="sxs-lookup"><span data-stu-id="ef413-147">The orchestration can be queried for the status or sent event notifications.</span></span>
