## <a name="synchronized-code-regions"></a><span data-ttu-id="2989a-163">Regiones de código sincronizado</span><span class="sxs-lookup"><span data-stu-id="2989a-163">Synchronized code regions</span></span>  

 <span data-ttu-id="2989a-164">Puede usar la clase <xref:System.Threading.Monitor> o una palabra clave del compilador para sincronizar bloques de código, métodos de instancia y métodos estáticos.</span><span class="sxs-lookup"><span data-stu-id="2989a-164">You can use the <xref:System.Threading.Monitor> class or a compiler keyword to synchronize blocks of code, instance methods, and static methods.</span></span> <span data-ttu-id="2989a-165">No se admiten los campos estáticos sincronizados.</span><span class="sxs-lookup"><span data-stu-id="2989a-165">There is no support for synchronized static fields.</span></span>  
  
 <span data-ttu-id="2989a-166">Visual Basic y C# admiten el marcado de bloques de código con una palabra clave de lenguaje concreta, la instrucción `lock` de C# o la instrucción `SyncLock` de Visual Basic.</span><span class="sxs-lookup"><span data-stu-id="2989a-166">Both Visual Basic and C# support the marking of blocks of code with a particular language keyword, the `lock` statement in C# or the `SyncLock` statement in Visual Basic.</span></span> <span data-ttu-id="2989a-167">Cuando se ejecuta el código en un subproceso, se realiza un intento de adquirir el bloqueo.</span><span class="sxs-lookup"><span data-stu-id="2989a-167">When the code is executed by a thread, an attempt is made to acquire the lock.</span></span> <span data-ttu-id="2989a-168">Si otro subproceso ya lo ha adquirido, el subproceso se bloquea hasta que el bloqueo vuelva a estar disponible.</span><span class="sxs-lookup"><span data-stu-id="2989a-168">If the lock has already been acquired by another thread, the thread blocks until the lock becomes available.</span></span> <span data-ttu-id="2989a-169">Cuando el subproceso sale del bloque de código sincronizado, el bloqueo se libera, independientemente de la manera en que lo haga.</span><span class="sxs-lookup"><span data-stu-id="2989a-169">When the thread exits the synchronized block of code, the lock is released, no matter how the thread exits the block.</span></span>

 <span data-ttu-id="2989a-172"><xref:System.Threading.Thread.Interrupt%2A?displayProperty=nameWithType> puede utilizarse para que un subproceso salga de operaciones de bloqueo, como esperar para acceder a una región de código sincronizado.</span><span class="sxs-lookup"><span data-stu-id="2989a-172"><xref:System.Threading.Thread.Interrupt%2A?displayProperty=nameWithType> can be used to break a thread out of blocking operations such as waiting for access to a synchronized region of code.</span></span> <span data-ttu-id="2989a-173">**Thread.Interrupt** también se utiliza para que los subprocesos salgan de operaciones como <xref:System.Threading.Thread.Sleep%2A?displayProperty=nameWithType>.</span><span class="sxs-lookup"><span data-stu-id="2989a-173">**Thread.Interrupt** is also used to break threads out of operations like <xref:System.Threading.Thread.Sleep%2A?displayProperty=nameWithType>.</span></span>  
  
> [!IMPORTANT]
> <span data-ttu-id="2989a-174">No bloquee el tipo, es decir, `typeof(MyType)` en C#, `GetType(MyType)` en Visual Basic o `MyType::typeid` en C++, para proteger los métodos `static` (métodos `Shared` en Visual Basic).</span><span class="sxs-lookup"><span data-stu-id="2989a-174">Do not lock the type — that is, `typeof(MyType)` in C#, `GetType(MyType)` in Visual Basic, or `MyType::typeid` in C++ — in order to protect `static` methods (`Shared` methods in Visual Basic).</span></span> <span data-ttu-id="2989a-175">Utilice en su lugar un objeto estático privado.</span><span class="sxs-lookup"><span data-stu-id="2989a-175">Use a private static object instead.</span></span> <span data-ttu-id="2989a-176">Asimismo, no utilice `this` en C# (`Me` en Visual Basic) para bloquear los métodos de instancia.</span><span class="sxs-lookup"><span data-stu-id="2989a-176">Similarly, do not use `this` in C# (`Me` in Visual Basic) to lock instance methods.</span></span> <span data-ttu-id="2989a-177">Utilice en su lugar un objeto privado.</span><span class="sxs-lookup"><span data-stu-id="2989a-177">Use a private object instead.</span></span> <span data-ttu-id="2989a-178">Una clase o instancia se puede bloquear por código distinto del suyo, lo que puede producir interbloqueos o problemas de rendimiento.</span><span class="sxs-lookup"><span data-stu-id="2989a-178">A class or instance can be locked by code other than your own, potentially causing deadlocks or performance problems.</span></span>
